# Конституция платформы IT Mentor | Сергей Жуков

## Core Principles

### I. Спецификация и измеримые исходы
Каждая фича стартует со spec/plan по шаблонам репозитория; юзер-стори обязаны быть независимыми, с проверяемыми критериями приемки и явными edge cases; метрики успеха фиксируются заранее и используются как целевые для проверки поставки.

### II. Test-First дисциплина
Тесты пишутся до реализации: юнит-тесты покрывают бизнес-логику, интеграционные требуются при выходе за границы модуля или работе с внешними системами; новая работа начинается с падающих тестов; CI должен запускать линтеры и тесты по умолчанию.

### III. Поставки слайсами и трассируемость
Работа доставляется независимыми инкрементами по юзер-стори; задачи маппятся на стори и не ломают их изоляцию; в PR обеспечивается трассировка story → tasks → код и возможность продемонстрировать ценность каждой стори отдельно.

### IV. Простота и наблюдаемость
Предпочитается наименьшая достаточная сложность; избегаем абстракций без явной выгоды; ключевые пути и ошибки покрываются структурированными логами и метриками, достаточными для диагностики регрессий и деградаций производительности.

### V. Безопасность и безопасные изменения
Секреты и чувствительные данные не хранятся в репозитории; минимизируем сбор/хранение PII; изменения пользовательских интерфейсов и контрактов сопровождаются планом миграции или сохраняют совместимость; применяем семантическое версионирование.

## Дополнительные ограничения
Секреты только через внешние хранилища/переменные окружения; проверяем зависимости на известные уязвимости перед вводом; для критичных операций фиксируем логи ошибок и латентности; PII собирается только при обосновании и с минимальным удержанием.

## Процесс разработки
Используем стандартные шаблоны spec/plan/tasks; Constitution Check выполняется перед исследованием (Phase 0) и повторно после дизайна; каждый PR включает доказательство прохождения тестов и соответствия принципам наблюдаемости и безопасности; задачи группируются по независимым юзер-стори.

## Governance
Настоящая конституция превосходит остальные гайды; изменения вносятся через PR с описанием влияния, планом миграции/коммуникаций и обязательным семантическим бампом: Major — пересмотр/удаление принципов, Minor — добавление или существенное расширение, Patch — уточнения без изменения смысла; при ревью проверяются тест-план, независимость стори, отсутствие секретов и покрытие наблюдаемости/безопасности.
